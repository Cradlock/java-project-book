## ООП

`Java` - сам по себе обьектно-ориентированный.
ООП на java похоже на `c++`.Те же классы обьект и т.д
---
Все строится на обьектах и классах.

```
class Name{
    ... поля

    ... методы
}

```



Также есть понятия модификатор ключевые слова которые определяют
уровень доступа и поведение.

## Модификаторы:
    
### Уровня доступа:

- `public`: Публичный доступ. работает для:
    * классов
    * полей
    * методов

- `protected`: Внутри пакета + наследникам в других пакетах.
работает для:
    * поля
    * метод

- `private`: Доступно только внутри своего класса .
работает для:
    * поля
    * методы

> *Доп.инфо* - Внутренние и вложенные классы могут быть private,
protected но внешние нет

### Поведения:

- `final`: Запрет на изменение
    * класс: Запрешает наследование

    * поле: Делает переменную константной 

    * метод: Запрешает переопределение

- `abstract`: Заготовка
    * класс: Нельзя создать обьект

    * метод: метод без тела.Просто нужен чтобы показать что нужно сделать

- `static`: Принадлежность к классу
    * поле: Общее поля для всех обьектов

    * метод: Общий метод(можно использовать без создания обьекта) 

- `synchronized`: Многопоточность
    * метод: Только один поток может выполнять метод одновременно

- `volatile`: Многопоточность
    * поле: Указывает то что значение может менятся разными потоками

- `transient`: Сериализация
    * поле: Почемает поле которое не нужно сохранять (сериализовать) 
    при записи обьекта в файл или передаче по сети

- `native`: Связь с С/С++
    * метод: обозначает что метод написан на другом языке и 
    подключается через `JNI` (Java Native interface - позволяет JVM 
работать с: С,С++,ASM)

- `strictfp`: Математика
    * метод/класс: Гарантирует одинаковую точность вычислений
на разных процессорах
