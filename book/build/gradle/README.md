## gradle

`Gradle` - это система сборки которая автоматически может собирать многофайловые проекты и подгружать зависимости.
---
Но `gradle` использует специальную структуру.

> Минимальный пример
```
src/
    ...

build.gradle 

```

`src` - код

`build.gradle` -  основной файл сборки 

## Структура
Gradle любитель порядка у него есть особая структура файлов для Java. 
Как было сказано весь код внутри папки `src`.
```
├── main
│   ├── java <- основной код
│   │   ├── common
│   │   │   └── Main.java
│   │   └── models
│   │       └── Product.java
│   │
│   └── resources <- ресурсы(манифесты,картинки,шрифты,настройки)
│           └── settings.txt 
│  
└── test <- Тесты проекта
```
В таком случае `build.gradle` может выглядет вот так.
```
plugins {
    id 'java'
    id 'application'
}

application {
    mainClass = 'common.Main'
}
```



- `plugins`: Это настройка ядра для проекта. Сам по себе gradle не знает 
как использовать Java, искать файлы собирать jar. 
    - `java`: Подключает Java 
    - `application`: Добавляет возможность упаковать все в приложени,добавляет
задачу `run`


## gradlew 
`./gradlew` - файд для поддержки системы версий в Gradle. Без него будет 
сложно использовать так как проект будет искать нужную версию на пк, а с gradlew сам подтянет
нужную версию. 

- `gradle wrapper`: создание gradlew файла 

> Он дает список доступных команд

* ./gradlew build          # Полная сборка проекта
* ./gradlew clean          # Удаление папки build (очистка)
* ./gradlew tasks          # Список всех доступных задач
* ./gradlew run            # Запуск


## shadowJar 
`Fat JAR` - инструмент помогающий скомпоновать все зависимости в один .jar 
```
plugins {
    id("com.gradleup.shadow") version "9.3.1"
}

```
Команда - `./gradlew shadowJar`




## Зависимости 
Чтобы скачивать зависимости нам нужно:

### Указать откуда 
```
repositories {
    mavenCentral()
}

```


### Что именно

* `plugins`
Указывает новые инструменты для `gradle` добавялет новые команды.  

* `dependencies`
Указывает какой Java библиотеки использовать это уже чистый код.
```
dependencies {
    implementation `зависимость`
}

```


### Кастомные настройки
Ну у каждого инструмента свои настройки









## Мультипроект
Можно сделать несколько модулей в каждом из которых свой build.gradle 

* Структура
```
MyProject
├── app
│   └── build.gradle
├── library
│   └── build.gradle
│
├── build.gradle 
└── settings.gradle

```
- `app/library`: Модули

- `settings.gradle`: Обшая настройка для всего проекта тут мы подключем модули 

- `build.gradle`: Общие модули для всего 

- `*/build.gradle`: Настройка конкретного модуля
