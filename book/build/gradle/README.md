## gradle

`Gradle` - это система сборки которая автоматически может собирать многофайловые проекты и подгружать зависимости.
---
Но `gradle` использует специальную структуру.

> Минимальный пример
```
src/
    ...

build.gradle 

```

`src` - код

`build.gradle` -  основной файл сборки 

## Структура
Gradle любитель порядка у него есть особая структура файлов для Java. 
Как было сказано весь код внутри папки `src`.
```
├── main
│   ├── java <- основной код
│   │   ├── common
│   │   │   └── Main.java
│   │   └── models
│   │       └── Product.java
│   │
│   └── resources <- ресурсы(манифесты,картинки,шрифты,настройки)
│           └── settings.txt 
│  
└── test <- Тесты проекта
```
В таком случае `build.gradle` может выглядет вот так.
```
plugins {
    id 'java'
    id 'application'
}

application {
    mainClass = 'common.Main'
}
```



- `plugins`: Это настройка ядра для проекта. Сам по себе gradle не знает 
как использовать Java, искать файлы собирать jar. 
    - `java`: Подключает Java 
    - `application`: Добавляет возможность упаковать все в приложени,добавляет
задачу `run`







## Зависимости 
Чтобы скачивать зависимости нам нужно:

### Указать откуда 
```
repositories {
    mavenCentral()
}

```


### Что именно

* `plugins`
Указывает новые инструменты для `gradle` добавялет новые команды.  

* `dependencies`
Указывает какой Java библиотеки использовать это уже чистый код.
```
dependencies {
    implementation `зависимость`
}

```


### Кастомные настройки
Ну у каждого инструмента свои настройки
